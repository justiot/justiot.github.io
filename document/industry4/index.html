技術發展

工業工廠自動化目前的主架構，還在工業3.0階段，即以電氣電子及電腦為主的思維所建構的系統，其雖有網路，但仍以電腦的自動化為主，網路局限在廠內網路。

近年來，各種工程技術，尤其是資訊科技，如雲端計算,大數據人工智慧,物聯網,嵌入式系統,虛擬/擴充實境,３Ｄ積層製造,機器人,網路安全,軟體工程等，進展神速。 為了將這些技術有效整合應用到製造業上，工業製造必須有一個結構性的調整，即所謂的工業4.0.

工業4.0的核心在建構網絡實體系統(Cyber-Physical System, CPS),其包括下示五個層次,

1.智慧連線(Connection):收集資料.(元件視野)
2.資訊提取(Conversion):將原始資料轉為有用資訊.(單一機器視野)
3.虛擬採礦(Cyber):系統模擬及機器學習進行深刻分析.(序列機器視野)
4.認知診斷(Cognition):即時監看,機器預測,健康診斷,故障預估.
5.智慧調適(Configuration):建立自調適系統.(企業視野).

目前技術

企業工業主架構在工業3.0階段,主要是上圖左方所示的五層架構,其在工業電腦的思維下,層層負責. 層與層之間的溝通,以OPC伺服器為中介.

以目前資訊技術的觀點來看,傳統的疊床架屋五層架構,效能不彰,在資訊技術的衝擊下,將走向扁平化; 資料收集放在雲端平台上,而各式功能,成為雲端平台上的服務,這期間,上端的ERP廠商推動雲端架構的形成,而底層智慧感測器控制器將資料送上雲端.(中間層的硬體商及軟體商可能成為輸家?)

OPC技術的設計,以一個工業標準的方式,包容工廠內各廠牌的異質裝置,將各式工控控制器感測器的資料放在OPC伺服器上,作為各階層(SCADA,MES,ERP)的資料站及中介站.

OPC技術以微軟的COM/DCOM技術為主,在現今的網路環境下有諸多重大缺點,如綁定微軟的Windows操作系統,網路安全很脆弱等.

業界聯盟,在資訊科技的衝擊下,推出新一代的標準 OPC UA,以承接OPC,希望將舊架構迎向新時代.

但如上圖所示,OPC與OPC UA底層通訊架構不同,新一代的OPC UA不相容於之前的傳統OPC, 其通訊必須組過中間裝置(Bridge, Gateway),讓以往疊床架屋的架構,更加紛亂. 其實,OPC UA技術就是Web技術,其提出是想保有固有市場的利益,或說是延續業者的投資.

技術服務

邁向工業4.0

本團隊提供的技術服務，可分為三個面向： 　　　　　　　　　　　

  1. 工業自動化、商業自動化、通信自動化和網路自動化建置:

    2. 引進國外的先進工業技術，開發、研發各種 資訊和自動化産品，並承接圖控、自動化系統專案工程.爲用戶提供具有高品質、高性能和低成本的系統整合方案.

  2. 設計建置工業4.0雲端平台:
    1. JustIoT物聯網:泛用的雲端物聯網平台
    2. OPC UA Server/Client: Node Open Source OPC UA(進行中)
    3. 機械學習(Machine Learning)平台: 泛用的機械學習平台(進行中)
  3. 引導現有的工業2.0,工業3.0系統,共同邁向工業4.0.
    1. OPC Bridge: 資料在OPC伺服器間轉移.
    2. OPC2PLC: 讀OPC資料到PLC暫存器.
    3. OPC2Just: 建立OPC伺服器與JustIoT物聯網雙向通訊.
    4. PLC2Just: 建立PLC(Modbus TCP)與JustIoT物聯網雙向通訊.
    5. 智慧感測卡: 資料自動抓取.
